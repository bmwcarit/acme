Example of a small ACME project

In order to demonstrate the usage of ACME, the small sample project "HelloACME" is presented here.

In the topmost project directory there is a CMakeList.txt with the following commands:


The command CMAKE_MINIMUM_REQUIRED determines the required CMake version. If the used version is older,
then the process will be stopped and an error will be printed. PROJECT specifies the name or the created workspace of the new project.
The INCLUDE command adds the used ACME script acme.cmake. This script looks for toolchain files, for plugins and performs the first configurations.
The command ACME_ADD_SUBDIRECTORY tells CMake, that there is a directory "modules" with an additional CMake_Lists.txt.
ACME_FINALIZE creates, if the cache variable CONFIG_BUILD_GLOBAL_TEST_EXECUTABLE has been set to "TRUE", a test executable and prints the "configuration report".













The CmakeList.txt in the directory "modules" contains only the command ACME_ADD_SUBDIRECTORY.
It tells CMake, that there exists the folders "Hello"and "Main" with additional CMake_Lists.txt.









This is an image of the CMakeList.txt of the module "Hello".

By means of the command ACME_ADD_MODULE the module properties will be set and loaded into the cache.
As moduletypes you can choose "static", "dynamic", "exe" or "tests".
ACME_ADD_FILE selects the used files. You have to state the filename without file extensions, ACME automatically
looks for source files and test files.
ACME_REQUIRED_PACKAGE indentifies, depending on the operating system, the package include directories and the package libraries.





The CMakeList.txt in the module "Main" looks very similar.
Here the command ACME_ADD_DEPENDENCY additionally adds a dependency to the module "Hello".
