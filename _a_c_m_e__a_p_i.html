ACME_ADD_CMAKE_PROJECT

Function that uses the commands ExternalProject_Add() and Add_External_Library() to add a cmake project.

Signature:

1 ACME_ADD_CMAKE_PROJECT( <project_name>
2  LIBNAMES "<libname1>" "<libname2>" ...
3  LIBDIRS "<libdir>" ...
4  INCLUDE_DIRS "<incl_dir>" ... (relative from "${PROJECT_SOURCE_DIR}/build/<project_name>")
5  BINARY_INCLUDE_DIRS "<incl_dir>" ... (relative from "${PROJECT_SOURCE_DIR}/build/<project_name>/src/<project_name>-build/"
6  ABSOLUTE_INCLUDE_DIRS "<incl_dir>" ... (absolute path)
7  URL "<url>"
8  CHECKSUM "<url_md5>"
9  SOURCE_DIR "<project_source_dir>"
10  CMAKE_ARGUMENTS "<argument_name>:<typ>=<value>" ...
11  DEPENDENT_DEFINITIONS "<definition1>" ...
12  DEPENDENT_DEBUG_DEFINITIONS "<definition2>" ...
13  DEPENDENT_RELEASE_DEFINITIONS "<definition3>" ...
14  INSTALL 1 | 0 (default)
15  )
Parameters
aap_project_namename of the added cmake project
Modifies ACME State Variables:
BUILD_${aap_project_name}
[${aap_project_name}_INCLUDE_DIR]
[${aap_project_name}_LIBRARIES_DIR]
[${aap_project_name}_LIBRARIES]
[${aap_project_name}_FOUND]
[${aap_project_name}_INTERNAL]

ACME_REQUIRED_PACKAGE

ACME_ADD_FILE

Adds all found public header files, intern header files and modules source files with the filename given in the
<parameter>. It additionally adds all testfiles with the filename <parameter>Test. To add files use just the
filename without any file extensions. To add several different files use several function calls with just 1
Parameter per call!

Parameters
${ARGN}name of the added public header files, intern header files, module source files and test files
Modifies ACME State Variables:
${CURRENT_MODULE_NAME}_HAS_SOURCE_FILES
${CURRENT_UPPER_MODULE_NAME}_MODULE_SOURCE_FILES
${CURRENT_MODULE_NAME}_TEST_FILES

ACME_LINK_LIBRARY

Adds the library ll_name to the adequate list, which is then linked to the target by the function
INTERNAL_JUST_DOIT(). The target is specified by the invoking CMakeList.txt. To add several libraries, you
can either use several function calls, or specify the arguments with spaces in between.

Parameters
ll_namename of the added library
${ARGN}list of names of additional libraries
Modifies ACME State Variables:
${CURRENT_MODULE_NAME}_LIBRARIES

ACME_LINK_LIBRARY_GROUP

Adds the library-group (indicated with spaces between the different libraries) to the adequate list, which is
then linked to the target by the function INTERNAL_JUST_DOIT(). The target is specified by the invoking
CMakeList.txt.

Parameters
llg_namename of the first library from the library group
${ARGN}names of the rest of the libraries
Modifies ACME State Variables:
${CURRENT_MODULE_NAME}_LIBRARIES

-----------------------------------------------------------------------------------------------------------------—

ACME_ADD_INSTALL_FILE

Adds the files in the paramters (separated by spaces) to the list ${CURRENT_MODULE_NAME}_INSTALL_FILES, which is
then used to install the files into the "bin"-directory (in the directory deliverable) by the function
INTERNAL_JUST_DOIT(). The CURRENT_MODULE_NAME is specified by the invoking CMakeList.txt. To install several
files, you can either use several function calls, or specify the arguments with spaces in between.

Parameters
${ARGN}names of the files that should be installed
Modifies ACME State Variables:
${CURRENT_MODULE_NAME}_INSTALL_FILES

ACME_FINALIZE

Function that removes duplicates from the global lists, builds unit tests (if the cache variable
CONFIG_BUILD_GLOBAL_TEST_EXECUTABLE is set to "True"), creates a dependency graph (if the cache variable
CONFIG_CREATE_DEPENDENCY_GRAPH is "True"), creates the build report (CONFIG_CREATE_BUILD_REPORT has to be set
to "True") and prints the build report.

-----------------------------------------------------------------------------------------------------------------—

ACME_PRINT_STATE

Prints all used cmake variables, build options, added cmake projects, used file extensions, external include
directories, used modules, module types, module properties, external library properties, package properties and
test properties. Use the function without paramters.

-----------------------------------------------------------------------------------------------------------------—

ACME_SAVE_STATE

Saves all used cache variables in the following manner:
<state_name>_<name_cache_variable>, whereby <state_name> is the parameter of ACME_SAVE_STATE() and
<name_cache_variable> is the name of the cache variable that is to be saved.

Parameters
state_namename of the saved state

-----------------------------------------------------------------------------------------------------------------—

ACME_DELETE_STATE

Deletes the state mentioned in the paramter of the function call.

Parameters
state_namename of the state that should be deleted

-----------------------------------------------------------------------------------------------------------------—

ACME_COMPARE_STATES

Compares state1 and state2 and prints the differences between these two states.
state1 and state2 have to be saved by the function ACME_SAVE_STATE() in advance.

Parameters
state1first state that ought to be compared
state2second state that ought to be compared

ACME_TEST

Tests the supplied ACME scripts. Currently there are 23 testcases implemented. To test the ACME scripts you just
need to configure the TestProject supplied with ACME (directory TestProject in the ACME source directory).
The results of the testcases will be printed. Delete your cache before testing the ACME scripts.
