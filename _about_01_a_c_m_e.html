Services of ACME

ACME offers a large number of services to the user:

  • a collection of useful ACME functions (see also section ACME_API)
  • cross compiling out of the box
  • integration of GoogleTest
  • integration of GoogleMock
  • a reasonable, given project structure
  • mechanisms to automatically build dependent projects
  • automatic development of the release
  • automatic generation of a build report
  • automatic generation of a dependency graph (which is integrated into the build report)
  • automatic generation of a code coverage report


Structure of an ACME project


This image shows the common project structure of an ACME project.
CMake files that are used to identify the needed packages, are saved into the subdirectory "cmake/modules".
Into the folder "plugins" you can deposit files that look for additional cmake projects.
The individual modules of your project should be saved into the "modules" folder.
They can consist of an "include" directory (to save the header files), a "res" folder (to save files of the ld command language), a "src" folder for the cpp files and an optional "test" directory for the test files.




















The deliverable structure is constructed as follows.
The deliverable folder of all built additional cmake projects are saved into the deliverable folder of the current project.
Every deliverable directory consists of a "bin" folder, where the debug and release executables are saved by ACME.
In an additional "doc" directory the generated build report (with dependency graph and code coverage) are deposited.
Furhtermore there exists a "lib" directory for the libraries and a "res" folder.

















Build report

If the cache variable CONFIG_CREATE_BUILD_REPORT is set to "TRUE" and the tool "Doxygen" (see section Installation) has been found,
ACME automatically creates a build report of the current projects. ACME will automatically look for "Doxygen" and print the current status.
You can find the build report by searching for the file index.html in "doc/html" of the deliverable folder of the built project.

This report contains a list of all classes, the class hierarchy and all include files.

Dependency graph

To integrate the dependency graph of your project into the above mentioned build report, set the cache variable CONFIG_CREATE_DEPENDENCY_GRAPH to "TRUE"
and install the tool "Graphviz" (see section Installation) into the standard directory of your operating systems. ACME will automatically look for this tool and print the current status.
Additionally ACME saves a copy of the dependency graph of the format png into the "doc" folder of the current deliverable directory.

Documentation

In order to provide project documentation, just add a Doxyfile.in in a folder named doc directly in your root directory. A documentation target will come up which builds doxygen documentation. If you put dox files in the folder as well, these will get included, too.
