What is ACME?

ACME is an extension to the open source build system CMake.
It provides several CMake scripts which help the users to build their projects. The provided ACME functions are invoked by the user
in their CMakeLists.txt. They set different cache variables, which are then committed to the CMake functions by the function ACME_FINALIZE.

Furthermore ACME provides additional information about the build to the user. Without additional effort the user gets a build report of the project. This report
contains a list of all classes, the class hierarchy, all include files. Furthermore it integrates a dependency graph of the used modules, external libararies and additional
CMake projects into the html build report.
Furthermore an analysis of the code coverage is executed by ACME.

Why is ACME useful?

ACME especially supports the developer at recurring activities when drafting a new project. It fulfils tasks like integrating tests, the configurations of flags,...
It supports cross compiling and different IDEs.
ACME is particularly helpful when creating dependencies to different projects.
